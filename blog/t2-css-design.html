Classes vs. IDs

January 5th, 2015

If you are new to web designing like I am, you probably aren't aware of the differences in using Classes versus IDs in CSS.

First off, Classes and IDs are merely HTML selectors or tags that need to be created or set by the user in order to use CSS to style.

IDs are unique, such that each page can have only one element with that ID and in addition, each element can have only one ID. To simply put it, your HTML/CSS code will show errors if a user uses the same ID on more than one element. It is best to only use IDs for any elements that are only used once on a given page.

Classes are common, meaning that a user can set the same class on multiple elements and in reverse, a user can use multiple classes on the same element. Classes have a wider range of application as they can reuse CSS styling elements and make your code more efficient.

A good example of Classes versus IDs is cars versus vehicle identification numbers (VIN). Toyota makes thousands and thousands of Toyota Corollas, however, each has their own VIN to identify a particular Corolla over the thousands being built. Toyota makes all the headlights of the Corolla same and doesn't differentiate headlights by VIN as that would be highly inefficient and costly. Classes are much like a common car that a company makes. Information that can be used again should be identified using classes. IDs are much like VINs. Information that is unique should be identified using IDs
