Ruby vs. JavaScript

Hashes vs. Objects

February 12th, 2015

What are Ruby Hashes? Ruby hashes are collections of key-value pairs. This is helpful when you need to map one value to another. You might want to map a set of items to information about the items. Here’s an example: 

      car = {
        “name” => “Porsche”,
        “year” = > 1999,
        “color” => “black”
        }

There are many ways to create a hash. 

      car = Hash.new
      car[“name”] = “Porsche”

You may access each named key. 

      car[:name]    #=> ‘Porche’

What are JavaScript Objects? JavaScript objects are similar to Ruby hashes such that they both are collections of pairs. In JavaScript, they are known as a collection of properties that have names and values. Here’s an example: 

      var car = {
      name: “Porsche”,
        year: 1999,
      color: “black”
      };

You may access object properties in two ways: 

      car.name;   //=> ‘Porsche’

or 

      car[name];  //=> ‘Porsche’
